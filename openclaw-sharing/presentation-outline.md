## 第1页：需求气泡图

### 视觉设计

```
        [信息获取]
       /    |    \
      /     |     \

      \     |     /
       \    |    /
        [项目开发]
              |
         [探索实验]
```

### 需求分类

| 大类 | 具体需求 | 关联 |
|------|---------|------|
| **信息获取** | 快速研究陌生领域、持续监控信息源、文献综述 | → 知识沉淀 |
| **知识沉淀** | 结构化笔记、长期记忆、知识库维护 | ← 信息获取，→ 项目开发 |
| **项目开发** | 写代码、调试、自动化脚本、工具开发 | ← 知识沉淀 |
| **日常协作** | 快速问答、日程管理、决策辅助 | ↔ 信息获取 |
| **探索实验** | 多 Agent 协作、A2A 通信、新技术尝鲜 | → 项目开发 |

### 讲解要点

- 这些需求不是独立的，是相互关联的
- 信息获取 → 知识沉淀 → 项目开发 是一个流程
- 不同需求适合不同的工具

## 第3页：Harness Engineering 概念

### 核心定义

> **Harness Engineering（驾驭工程）**：通过配置、定制、协作，让 AI Agent 成为工作流的有机组成部分，而非一次性工具。

### 三个层次

```
Level 1: 即用即走
├── 直接对话，完成任务
├── 无配置、无定制
└── 例：问 Hermes 一个问题，得到答案

Level 2: 配置适配
├── 加载 Skills、配置 Memory
├── Agent 理解你的偏好、背景
└── 例：Hermes 记住你的工作习惯

Level 3: 工作流嵌入
├── Agent 参与多阶段流程
├── 与其他 Agent 协作、自动化
├── 长期积累、持续优化
└── 例：OpenClaw + 2nd_brain + cron + A2A
```

### 项目示例对应

| 项目 | Harness Level | 说明 |
|------|--------------|------|
| 快速研究陌生领域 | Level 1 | 即用，完成即止 |
| 2nd_brain 维护 | Level 3 | 嵌入长期工作流，自动同步 |
| ClawTeam 开发 | Level 2-3 | 配置 + 多 Agent 协作 |
| A2A 通信 | Level 3 | 双 Agent 嵌入协作流程 |

### 讲解要点

- 不是所有项目都要 Level 3
- 不同需求适合不同层次
- Harness Engineering 的核心：**让 Agent 从工具变成伙伴**

### 关键问题引入

> "三个工具各有擅长的 Harness Level，这就是为什么我需要多个 Agent"

## 第4页：三个 Agent 工具对比表

### 详细对比

| 维度 | OpenClaw | Hermes Agent | Claude Code |
|------|----------|--------------|-------------|
| **定位** | 本地化定制 Agent 平台 | 云端通用 Agent | 代码专用 Agent |
| **Harness Level** | Level 1-3 全支持 | Level 1-2 | Level 1-2（代码场景） |
| **部署方式** | 本地部署 | 云端服务 | 云端服务 |
| **定制能力** | 高（Skills/Extensions/Memory） | 中（Skills/Memory） | 低（项目上下文） |
| **多 Agent 协作** | ✅ A2A、ClawTeam | ❌ 单 Agent | ❌ 单 Agent |
| **自动化能力** | ✅ Cron、定时任务 | ❌ | ❌ |
| **代码能力** | 中 | 中 | ✅ 强 |
| **研究能力** | 中 | ✅ 强（搜索/整理） | 中 |
| **长期积累** | ✅ 本地 Memory | ✅ 云端 Memory | 项目内 |
| **学习成本** | 高 | 低 | 低 |
| **数据安全** | ✅ 本地控制 | 云端存储 | 云端存储 |
| **适合场景** | 长期项目、自动化、多 Agent | 快速任务、研究、问答 | 开发、写代码 |

### 一句话定位

| 工具 | 一句话定位 |
|------|-----------|
| **OpenClaw** | 你的长期 AI 伙伴，可定制、可扩展、可协作 |
| **Hermes Agent** | 你的云端助手，快速响应、即用即走 |
| **Claude Code** | 你的 AI 程序员，专注代码开发 |

### 选择逻辑图

```
任务来了 → 判断类型：

快速问答/研究？ → Hermes
写代码/开发？ → Claude Code
长期积累/自动化/多Agent？ → OpenClaw
```

### 讲解要点

- 不是"哪个更好"，而是"哪个更适合"
- 三者可以配合使用（比如 Hermes 研究 → Claude Code 写代码 → OpenClaw 长期维护）

## 第5页：结语

### 核心信息

```
1. AI Agent 不是魔法，是工具
2. 不同需求需要不同的 Harness Level
3. 选择工具的关键：理解你的需求类型
4. 从 Level 1 开始，逐步深入
5. 让 Agent 从工具变成伙伴
```

### 建议听众思考的问题

> "你日常有哪些重复性工作？哪些信息获取任务？哪些可以交给 Agent？"

### 一句话收尾

> "我今天的分享不是为了推销工具，而是分享一种可能性：AI Agent 可以成为你工作流的一部分，关键是找到适合你的方式。"

|------|------|
| 讲太多技术细节 | 听众可能跟不上 |

- Agent 会犯错（需要验证）
- 不是万能的（有些事还是人做）
- 有学习成本（需要适应对话式交互）
- OpenClaw 本地部署有门槛
