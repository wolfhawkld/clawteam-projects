# KG RAG 意图属性建模研究

**创建时间**: 2026-04-01
**发起人**: Damon
**研究背景**: Content端意图表征强化，提升 ReAct agent + KG RAG + 用户反馈系统

---

## 研究目标

基于人机知识《RAG意图对齐的双向融合》，深入研究如何为 KG RAG 数据添加意图属性，以实现 Content 端意图表征强化。

## 核心问题

### 问题1: 意图属性的维度规划
KG RAG数据如果要加上意图属性，应该以什么维度或什么集合来规划这个意图属性？

**约束条件**:
- 不能仅从语义内容聚类（演绎式白名单）
- 需要支持动态演化
- 需要与现有 KG 结构兼容

### 问题2: 意图层面的归纳聚类
如果仅从RAG数据的语义内容进行聚类，本质上还是演绎法式的白名单构建。
如何设计一层"归纳法"式的意图层面聚类？

**关键洞察**:
- 语义聚类 ≠ 意图聚类
- 需要从"文档能回答什么问题"角度出发
- 可能需要反向生成意图标签

### 问题3: 用户反馈的意图关联
用户的反馈配合上下文能否作用于意图属性构建？如何实现？

**考虑方向**:
- 显式反馈（点赞/点踩、修正）→ 直接意图信号
- 隐式反馈（停留时间、重试次数）→ 间接意图信号
- 上下文关联 → 意图传播

---

## 系统上下文

当前系统架构：
- ReAct agent 作为推理核心
- KG RAG 作为知识检索层
- 用户反馈作为质量信号

期望改进：
- 在 Content 端注入意图信息
- 实现 Query-to-Intent 匹配（而非 Query-to-Document）
- 利用用户反馈优化意图表征

## 研究产出要求

1. 意图属性的形式化定义
2. 意图聚类的算法设计
3. 用户反馈与意图属性的关联机制
4. 可落地的技术方案
