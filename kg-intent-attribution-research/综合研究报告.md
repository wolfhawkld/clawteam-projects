# KG RAG 意图属性建模研究 - 综合报告

**研究时间**: 2026-04-01
**研究方法**: ClawTeam 多 Agent 协作研究
**项目路径**: `clawteam-projects/kg-intent-attribution-research/`

## 研究概述

基于人机知识《RAG意图对齐的双向融合》的核心洞察——**Content端意图表征强化是被忽视但极具潜力的研究方向**，本研究深入探索了三个核心问题：

1. KG RAG数据意图属性的维度规划
2. 从语义聚类到意图归纳的方法设计
3. 用户反馈与意图属性的关联机制

---

## 核心发现汇总

### 问题1: 意图属性维度规划

**结论**: 提出适配KG结构的层次化意图属性体系

| 层级 | 意图类型 | 说明 |
|-----|---------|------|
| **顶层** | QueryIntent / ActionIntent / MetaIntent | 三大意图类别 |
| **二级** | EntityQuery / RelationQuery / AttributeQuery / ComplexQuery | 查询子类型 |
| **属性** | 8核心维度 + 4补充维度 | hop_count, answer_type, complexity_level等 |

**关键创新**:
- 组合意图支持（sub_intents数组）
- KG元素绑定（kg_entities字段）
- JSON Schema形式化定义（可直接落地）

详见: `report-intent-dimensions.md`

---

### 问题2: 意图归纳聚类方法

**结论**: 提出"混合策略"的四阶段意图聚类框架

```
Phase1: AQ问题生成 + KG三元组增强
Phase2: UMAP降维 → HDBSCAN聚类 → 发现意图簇
Phase3: LLM生成簇标签
Phase4: 层次聚类 → L1/L2/L3层级构建
```

**核心洞察**:
| 维度 | 演绎式（白名单） | 归纳式 |
|-----|---------------|-------|
| 意图来源 | 预定义 | 从数据涌现 |
| 新意图 | 人工添加 | 自动发现 |
| 可扩展性 | 受限 | 开放 |

**推荐方案**: AQ问题生成 + QAEncoder期望估计 + HDBSCAN聚类组合

详见: `report-intent-clustering.md`

---

### 问题3: 用户反馈与意图关联

**结论**: 设计三层反馈-意图映射机制

```
反馈信号 → 信号聚合 → 意图推断 → 意图属性更新
```

**反馈权重设计**:
| 反馈类型 | 意图信号 | 权重系数 |
|---------|---------|---------|
| 点赞 | 意图匹配成功 | α = +0.3 |
| 点踩 | 意图匹配失败 | α = -0.5 |
| 答案修正 | 精准意图定位 | α = +1.0 |

**关键借鉴**:
- **Pistis-RAG**: Listwide反馈模型（整列表反馈更有效）
- **RLUF**: 多目标对齐框架（Love Reaction提升28%）
- **CID-GraphRAG**: 意图转换图 + 双路径检索（质量提升57.9%）

详见: `report-feedback-intent.md`
