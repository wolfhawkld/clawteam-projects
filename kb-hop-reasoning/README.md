## 核心结论

### ⚠️ 战略洞察

> **KG-RAG/多跳技术不是提升LLM问答性能的唯一解，甚至在大多数场景下不是最优解。**

向量RAG + 强LLM 已经能解决 **80%+** 的问答需求。KG-RAG 仅在以下场景值得投入：
- 多跳问题占比 > 40%
- 需要可审计推理链（医疗、金融、法律）
- 已有或可低成本构建知识图谱

## ⭐ 关键工程决策

### 决策1：权重维护粒度

> **核心洞察**：权重必须与语义上下文绑定，否则无法支持有效的多跳推理。

**方案**：语义聚类级权重 + 三层权重体系

详见：[weight-maintenance-design.md](./weight-maintenance-design.md)

---

### 决策2：动态意图策略架构 ⭐ 最终方案

> **核心思想**：以语义向量为核心，意图策略驱动，动态扩展权重项，按策略召回排序

```
用户问题 → 意图识别 → 策略匹配 → 权重计算 → 排序召回
                ↓
         意图策略池
         ├─ 事实查询策略
         ├─ 多跳推理策略
         ├─ 时效敏感策略
         └─ ... (可动态扩展)
```

**优势**：
- ✅ 语义优先：语义向量作为核心表示
- ✅ 策略驱动：不同意图使用不同权重策略
- ✅ 动态扩展：新意图策略可运行时注册
- ✅ 可组合：多策略组合，权重可叠加

详见：[dynamic-strategy-architecture.md](./dynamic-strategy-architecture.md)
