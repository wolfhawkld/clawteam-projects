## 1. 研究定位

### 1.1 核心理念

> **多跳推理不应停留在"路径遍历"，而应上升到"因果推理"**

当前知识图谱多跳推理研究存在关键问题：

| 问题 | 描述 |
|-----|------|
| **相关性 ≠ 因果性** | 图中的边往往表示关联，而非因果关系 |
| **路径≠推理** | 找到路径不等于完成推理，缺乏逻辑验证 |
| **缺乏可解释性** | 黑盒模型难以解释"为什么这样推理" |

**解决方向**：将因果推理引入多跳推理，构建真正的"推理链"而非"遍历链"。

### 1.2 研究价值

| 维度 | 传统多跳推理 | 因果推理增强 |
|-----|-------------|-------------|
| **推理质量** | 基于相关性，可能误导 | 基于因果性，逻辑可靠 |
| **可解释性** | 黑盒路径 | 可追溯因果链 |
| **泛化能力** | 依赖图结构 | 因果规律可迁移 |
| **应用场景** | 通用检索 | 法律、医疗、金融等高可信场景 |

## 2. 因果推理与多跳推理的关系

### 2.1 概念对比

| 概念 | 定义 | 多跳推理中的体现 |
|-----|------|-----------------|
| **关联** | X与Y同时出现 | 图中的边 (X → Y) |
| **因果** | X导致Y发生 | 需要额外建模 |
| **干预** | 如果改变X，Y会怎样 | What-if推理 |
| **反事实** | 如果X没发生，Y会怎样 | 假设性推理 |

### 2.2 从"路径"到"因果链"

**传统多跳推理**:
```
问题: "A公司破产会影响B公司吗？"
路径: A公司 → 合作关系 → B公司
问题: 这条路径说明什么？因果关系？仅仅是关联？
```

**因果推理增强**:
```
问题: "A公司破产会影响B公司吗？"
因果链: A公司破产 → (供货中断) → B公司生产受阻 → (收入下降) → B公司经营困难
关键: 每一步都有因果机制解释
```

### 2.3 研究问题转化

| 传统问题 | 因果视角问题 |
|---------|-------------|
| 找到A到B的路径 | A是否因果影响B？ |
| 路径长度越短越好 | 因果链是否合理？ |
| 所有路径加权融合 | 哪条因果链最可信？ |
| 基于历史数据预测 | 因果效应大小是多少？ |

## 3. 核心研究方向

### 3.1 方向一：因果知识图谱构建

**目标**: 从普通知识图谱构建因果知识图谱

**关键任务**:

| 任务 | 方法 | 产出 |
|-----|------|------|
| 因果边识别 | 因果发现算法 (PC, FCI, NOTEARS) | 标注因果/关联边 |
| 因果强度估计 | 因果效应估计 (倾向得分, 双重差分) | 边权重 |
| 因果机制建模 | 结构因果模型 (SCM) | 因果机制描述 |

**技术路线**:
```
普通知识图谱
    │
    ↓ 因果发现
候选因果边
    │
    ↓ 因果验证
因果知识图谱
    │
    ↓ SCM建模
可推理因果模型
```

---

### 3.2 方向二：因果链推理方法

**目标**: 基于因果链的多跳推理算法

**核心创新点**:

1. **因果路径发现**
   - 不仅仅是图路径，而是因果有效路径
   - 引入d-分离、后门准则等因果推断工具

2. **因果链验证**
   - 每一步推理都有因果机制支撑
   - 引入反事实检验

3. **因果效应量化**
   - 不仅回答"是否影响"，还要回答"影响多大"
   - 引入因果效应估计方法

**方法框架**:
```
┌─────────────────────────────────────────────────────────────┐
│                    因果链推理框架                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   问题分析                                                  │
│       │                                                     │
│       ↓                                                     │
│   ┌─────────────────┐                                       │
│   │ 因果路径发现    │ ← d-分离、后门准则                    │
│   └────────┬────────┘                                       │
│            │                                                │
│            ↓                                                │
│   ┌─────────────────┐                                       │
│   │ 因果链验证      │ ← 反事实检验、机制解释                │
│   └────────┬────────┘                                       │
│            │                                                │
│            ↓                                                │
│   ┌─────────────────┐                                       │
│   │ 因果效应估计    │ ← 倾向得分、工具变量                  │
│   └────────┬────────┘                                       │
│            │                                                │
│            ↓                                                │
│   答案 + 置信度 + 因果解释                                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 3.3 方向三：应用场景

**高价值场景**:

| 场景 | 因果问题 | 价值 |
|-----|---------|------|
| **法律推断** | 条款A是否导致结论B？ | 判例推理、合同有效性 |
| **医疗诊断** | 症状A是否由疾病B引起？ | 诊断推理、治疗方案 |
| **金融风控** | 事件A是否导致风险B？ | 风险传导、压力测试 |
| **供应链** | 中断A是否影响供应B？ | 风险预警、替代方案 |

**示例：法律推断**:
```
问题: "根据合同法，该合同是否有效？"

传统多跳推理:
  合同条款 → 合同法第X条 → 有效/无效
  问题: 为什么这条法律适用？缺乏解释

因果链推理:
  合同条款分析
      ↓ (因果: 条款包含无效情形)
  匹配合同法第52条 (无效合同情形)
      ↓ (因果: 法律规则适用)
  因果链: 条款X违反法律强制规定 → 触发第52条 → 合同无效
      ↓ (反事实检验)
  如果条款X不存在，合同是否有效？→ 检验因果性
      ↓
  结论 + 法律依据 + 推理链解释
```

## 4. 关键技术

### 4.1 因果发现方法

| 方法 | 类型 | 适用场景 | 局限性 |
|-----|------|---------|--------|
| **PC算法** | 约束型 | 观测数据 | 假设无隐藏混淆 |
| **FCI算法** | 约束型 | 有隐藏混淆 | 计算复杂 |
| **NOTEARS** | 优化型 | 连续优化 | 可扩展性 |
| **GNN-因果发现** | 深度学习 | 图结构数据 | 需要大量数据 |

### 4.2 因果效应估计

| 方法 | 适用场景 | 关键假设 |
|-----|---------|---------|
| **倾向得分匹配** | 观测数据 | 无未观测混淆 |
| **双重差分** | 面板数据 | 平行趋势 |
| **工具变量** | 有内生性 | 工具变量有效性 |
| **断点回归** | 政策评估 | 局部随机化 |

### 4.3 因果推理与大模型结合

| 方向 | 方法 | 优势 |
|-----|------|------|
| **LLM辅助因果发现** | 从文本提取因果关系 | 利用领域知识 |
| **因果增强RAG** | 因果链作为检索依据 | 提升推理质量 |
| **可解释生成** | 因果链作为解释依据 | 增强可信度 |

## 6. 与 intent-clustering 项目的关系

### 6.1 边界划分

| 项目 | 聚焦 | 不涉及 |
|-----|------|-------|
| **intent-clustering** | 意图识别、聚类、场景适配 | 多跳推理、因果推理 |
| **kb-hop-reasoning** | 多跳推理、因果推理 | 意图聚类 |

### 6.2 协同关系

```
用户问题
    │
    ▼
intent-clustering 项目
    │ 意图识别 → 场景分类
    │
    ├─ 事实查询 → 语义召回
    ├─ 探索检索 → 聚类召回
    └─ 因果推理问题 → kb-hop-reasoning 项目
                           │
                           ▼
                     因果链推理
                           │
                           ▼
                       推理结果
```

### 6.3 未来整合

两个研究方向成熟后，可整合为完整的**意图驱动因果推理系统**：
- intent-clustering: 决策层（意图识别、场景路由）
- kb-hop-reasoning: 执行层（因果链推理）
