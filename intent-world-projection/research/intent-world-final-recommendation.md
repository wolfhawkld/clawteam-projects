## 一、推荐路径 (Recommended Path)

### ✅ 采用方案A：混合式神经符号架构

**核心架构**：
- **神经网络层**：感知与模式识别（基于LLM预训练）
- **符号系统层**：推理与可解释性（知识图谱+规则引擎）
- **接口层**：双向语义转换

**意图召回架构**（由delivery-planner设计）：
```
┌─────────────────────────────────────────────────────────────┐
│                    用户输入 → 意图识别                      │
├─────────────────────────────────────────────────────────────┤
│  Layer 1: 快速识别层 (<20ms)                                │
│  - 关键词匹配、意图模板、用户画像召回                        │
├─────────────────────────────────────────────────────────────┤
│  Layer 2: 语义理解层 (20-50ms)                              │
│  - 向量检索、语义相似度、上下文依赖                         │
├─────────────────────────────────────────────────────────────┤
│  Layer 3: 深度推理层 (50-100ms)                             │
│  - 知识图谱关联、多维度融合评分、可解释推理                 │
└─────────────────────────────────────────────────────────────┘
```

**五维度召回**：
1. 语义相似度 (Vector Search)
2. 意图类别 (Intent Classification)
3. 用户画像 (User Profile)
4. 上下文依赖 (Context Dependency)
5. 知识关联 (Knowledge Graph)

### 推荐理由

| 维度 | 评分 | 说明 |
|------|------|------|
| 理论可行性 | ⭐⭐⭐⭐ | 符号落地问题有路径，具身认知理论支持 |
| 工程可行性 | ⭐⭐⭐ | 12-18月达生产级，架构复杂度可控 |
| 可解释性 | ⭐⭐⭐⭐⭐ | 符号层提供透明推理路径 |
| 实时性 | ⭐⭐⭐⭐ | 分层架构满足<100ms约束 |
| 中文适配 | ⭐⭐⭐ | 需跨语言迁移+数据标注投入 |

## 二、被拒绝的替代方案 (Rejected Alternatives)

### ❌ 方案B：纯神经网络架构

**拒绝理由**：
1. **可解释性差** - 黑盒模型无法提供透明推理
2. **语义落地未解** - 纯向量表示无法解决符号落地问题
3. **幻觉风险** - 无符号约束，意图推理不可控
4. **企业场景不适用** - 商业决策需要可追溯的推理链

**适用场景**：仅作为快速原型验证，不作为生产方案

### ❌ 方案C：纯符号推理架构

**拒绝理由**：
1. **灵活性差** - 规则维护成本高，难以适应新意图
2. **与现代AI脱节** - 无法利用LLM的语义理解能力
3. **知识获取瓶颈** - 需大量人工构建知识图谱
4. **开发周期过长** - 12-15月且风险高

**适用场景**：仅作为可解释性增强组件，嵌入方案A
