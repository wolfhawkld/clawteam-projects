## 🔑 对 RAG 优化方案的关键启示

### 启示1: 意图识别准确率是成败关键

```
意图分类准确率 → 直接影响检索策略 → 决定最终效果

建议:
- 意图识别准确率要求 >95%
- 使用 BERT/DeBERTa 微调 + 对比学习增强
- 边界意图需要人工兜底机制
```

### 启示2: 多意图处理是实际场景的常态

```
用户查询往往包含多个意图:
- "比较iPhone和Samsung的价格和性能" → 比较意图 + 多属性
- "退款并且要发票" → 退款意图 + 发票意图

解决方案:
1. LLM分解复杂查询为子查询
2. 并行检索 + 结果融合 (RRF)
3. 层次化处理（主要意图优先）
```

### 启示3: 意图-策略映射是核心架构

```
你的动态策略架构可以完美落地:

意图识别 → 策略匹配 → 权重计算 → 召回排序

具体实现:
1. 预定义意图策略 (事实查询、多跳推理、时效敏感等)
2. 意图向量匹配策略向量
3. 策略对应的权重项动态计算
4. 按策略排序召回
```

### 启示4: 表示学习决定聚类质量

```
意图向量表示质量 → 聚类效果 → 新意图发现能力

推荐技术栈:
- SimCSE / ProtoCL: 对比学习增强表示
- DAC / KNN-CL: 深度聚类方法
- OOS检测: 支持开放意图识别
```

## 📊 技术选型建议

### 意图分类模型选择

| 场景 | 推荐方案 | 准确率 | 成本 |
|------|---------|--------|------|
| 小规模/快速原型 | BERT-base 微调 | 93-94% | 低 |
| 高精度要求 | DeBERTa-v3 + SimCSE | 95%+ | 中 |
| Few-shot/新领域 | GPT-4 In-context | ~93% | 高 |
| 开放意图检测 | ADB / Granular-Ball | 95%+ | 中 |

### 聚类算法选择

| 场景 | 推荐方法 | 优势 |
|------|---------|------|
| 已知意图数 | K-Means++ + SBERT | 简单高效 |
| 未知意图数 | DAC / 层次聚类 | 自动确定 |
| 开放意图发现 | ProtoCL / CDI | OOS能力强 |
| 高精度要求 | KNN-CL / Granular-Ball | 性能最优 |
