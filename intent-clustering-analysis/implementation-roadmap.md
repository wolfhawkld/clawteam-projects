### 3.2 场景-策略映射

| 场景类型 | 特征 | 推荐策略 |
|---------|------|---------|
| **事实查询** | 单点信息、明确答案 | 语义召回 |
| **探索性检索** | 多实体对比、信息聚合 | 意图聚类召回 |
| **开放问答** | 广泛信息、无明确边界 | 混合召回 |

### 3.3 检索策略详解

**策略1: 语义相似召回**
```
原理: 查询与文档的语义向量相似度
适用: 事实查询、简单问答
实现: DPR/BGE + 向量索引
```

**策略2: 意图聚类召回**
```
原理: 查询归属意图簇 → 优先召回同簇文档
适用: 探索性检索、相似问题聚合
实现: HDBSCAN聚类 + 簇内检索
```

**策略3: 混合召回**
```
原理: 语义召回 + 意图召回加权融合
适用: 复杂场景、边界模糊场景
实现: RRF融合 + 动态权重
```

## 10. 总结

### 10.1 研究定位

> **聚焦意图聚类与动态检索，不涉及多跳推理**

本研究专注于：
- 意图识别与聚类方法
- 实体增强的聚类质量提升
- 用户反馈驱动的权重学习

### 10.2 核心创新

1. **实体增强意图聚类** — 融合知识图谱实体信息提升聚类质量
2. **场景-策略映射框架** — 根据问题特征自动选择检索策略
3. **反馈驱动权重学习** — 用户反馈优化策略权重

### 10.3 与 kb-hop-reasoning 的关系

| 项目 | 聚焦 | 边界 |
|-----|------|------|
| **intent-clustering** | 意图聚类 + 场景适配 | 不涉及多跳推理 |
| **kb-hop-reasoning** | 多跳推理 + **因果推理** | 独立研究方向 |

两个方向各有深度，可分别产出论文，未来可整合为完整系统。
